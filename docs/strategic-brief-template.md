# TennisSearch Strategic Brief Template

Use this template at the start of each iteration. Keep it short. The goal is to set direction, not prescribe implementation.

## 1. Iteration Goal

One sentence only.

Example:

`Increase the share of users who go from onboarding to a confirmed game.`

## 2. Primary Product Priority

Choose one:

- first confirmed game
- conversion from discover to response
- reliability and stability
- retention and repeat play
- recommendation quality
- premium conversion

## 3. Success Metric

Choose 1-2 metrics only.

- more created searches
- more responses to searches
- more accepted game requests
- more confirmed games
- fewer chat/notification failures
- fewer critical bugs in core flow

## 4. Guardrails

List what must not get worse.

- do not break onboarding
- do not break discover
- keep iOS and Web functionally aligned
- avoid unnecessary product complexity
- do not introduce risky schema changes without clear value

## 5. Product Principles For This Iteration

Choose up to 3.

- every screen should move the user toward a real game
- speed is more important than feature depth
- stability is more important than shipping extra scope
- recommendations must stay explainable
- avoid UI clutter
- keep flows simple for new users

## 6. Strategic Context

Optional short paragraph:

- what changed recently
- what concern you want the team to watch
- where you want the team to proactively find improvements

## 7. Explicit Non-Goals

Optional. Up to 3 bullets.

- no premium expansion this iteration
- no large redesign of profile
- no new sports beyond current support

## 8. Output Expected From The Team

The team should return:

- top opportunities found
- what they propose to improve now
- expected product impact
- implementation status for Web, iOS, backend, matching if relevant
- risks and follow-ups

## Ready-To-Use Example

```md
# Strategic Brief

## Iteration Goal
Increase the number of users who reach a confirmed game from the main search flow.

## Primary Product Priority
first confirmed game

## Success Metric
- more accepted game requests
- more confirmed games

## Guardrails
- do not break onboarding
- keep iOS and Web functionally aligned
- stability is more important than extra scope

## Product Principles For This Iteration
- every screen should move the user toward a real game
- avoid UI clutter
- recommendations must stay explainable

## Strategic Context
The team should proactively identify friction in discover, search creation, and chat-to-game conversion.

## Explicit Non-Goals
- no premium work
- no large visual redesign
```
